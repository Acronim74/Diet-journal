# Марафон −10 · PWA-дневник похудения

Однофайловое PWA-приложение для участников марафона похудения.

- Меню недели и список продуктов задаёт **коуч** — правит только `data.json`.
- Замеры (вес, шаги, талия / бёдра / грудь) ведёт **участник** — данные хранятся в `localStorage` устройства.
- Работает **офлайн** благодаря Service Worker (`sw.js`).
- Полностью устанавливается на телефон как нативное приложение (PWA).

## Структура проекта

```
/
├── index.html        ← всё приложение (HTML + CSS + JS в одном файле)
├── manifest.json     ← PWA-манифест
├── sw.js             ← Service Worker (офлайн-кэш)
├── data.json         ← КОУЧ РЕДАКТИРУЕТ ТОЛЬКО ЭТОТ ФАЙЛ
├── icons/
│   ├── icon-192.png
│   └── icon-512.png
├── scripts/
│   └── make-icons.ps1   ← регенерация иконок (PowerShell, Windows)
└── README.md
```

## Что внутри (4 вкладки + скрытый «Режим коуча»)

1. **Замеры** — прогресс марафона (кружки дней + прогресс-бар), форма ввода на сегодня, мини-график веса (SVG), итоговые показатели «ушло кг / ушло см».
2. **Меню** — переключатель недель + аккордеоны по дням и реальным датам; сегодняшний день раскрыт по умолчанию. Внутри дня — отдельный аккордеон для каждого приёма пищи (🌅 завтрак, ☀️ обед, 🍎 перекус, 🌙 ужин). Тап по блюду раскрывает состав, рецепт и КБЖУ.
3. **Список продуктов** — недельный shopping-list коуча + автоингредиенты, посчитанные по введённым в режиме коуча блюдам (с агрегацией одинаковых строк), + личные добавления участника.
4. **Архив** — таблица всех дней марафона, архив меню и списков продуктов по неделям, экспорт в PDF (через `window.print`), JSON-бэкап и импорт.

Плюс модальный онбординг при первом запуске (ввод имени) и опциональные напоминания через Notification API.

### Режим коуча

Скрытая страница для заполнения меню по датам. Открывается одним из трёх способов:

1. 5 быстрых тапов по заголовку «Марафон −10» в шапке;
2. Маленькая ссылка «Режим коуча» в самом низу вкладки **Архив**;
3. Прямая ссылка `…/Diet-journal/#coach`.

В любом случае появляется модалка с полем «код» — нужно ввести `2026` (значение константы `COACH_CODE` в `index.html`, поменять можно в одной строке).

В режиме коуча заполняется форма:

- **Дата** + **приём пищи** (завтрак / обед / перекус / ужин);
- **Название блюда**;
- **Ингредиенты** — добавляются по одному (название + вес: `50 г`, `200 мл`, `1 шт`);
- **Рецепт**, **КБЖУ**, **калории** — необязательно.

Все блюда сохраняются в `localStorage` под ключом `coach_overrides` и сразу появляются:

- на вкладке **Меню** соответствующей даты — как раскрывающаяся карточка блюда с полным составом;
- в **Списке продуктов** соответствующей недели — с пометкой «из меню» (одинаковые ингредиенты с одинаковой единицей измерения суммируются: `Овсянка — 50 г` + `Овсянка — 50 г` = `Овсянка — 100 г`).

Внизу страницы — список всех уже заполненных блюд (с возможностью «Изм.» или удалить), кнопки «Скачать JSON» (бэкап) и «Удалить все».

#### Публикация для всех участников

По умолчанию `coach_overrides` лежит **только в `localStorage` коуча** — другие участники его не видят. Чтобы поделиться меню со всеми, есть встроенная публикация в `data.json` через GitHub API.

1. **Один раз получите Personal Access Token (fine-grained):**
   - GitHub → ваш аватар → **Settings** → **Developer settings** → **Personal access tokens** → **Fine-grained tokens** → **Generate new token**.
   - Token name: `Diet-journal coach` (любое).
   - **Resource owner**: ваш аккаунт.
   - **Repository access**: *Only select repositories* → выбрать `Diet-journal`.
   - **Repository permissions** → **Contents**: *Read and write*.
   - Сгенерировать → скопировать строку `github_pat_…` (показывается один раз).
2. **Сохранить токен в приложении:**
   - В режиме коуча есть карточка **«Публикация для всех участников»**.
   - Вставьте токен → **«Сохранить токен»**. Токен хранится только на этом устройстве (`localStorage.gh_token`) и не попадает в экспорт JSON.
3. **Опубликовать:** нажмите **«Опубликовать»**. Приложение возьмёт текущий `data.json` из репозитория, заменит в нём поле `coach_overrides` вашими блюдами и закоммитит обратно. У всех участников меню обновится в течение 1–2 минут.

Если форкнули репозиторий — поменяйте константу `REPO` в начале `<script>` в `index.html`:

```js
const REPO = { owner: "Acronim74", name: "Diet-journal", branch: "main", file: "data.json" };
```

> **Архитектурный момент.** Когда устройство один раз вошло в режим коуча, выставляется `localStorage.coach_mode_active = "1"`. С этого момента приложение на этом устройстве рендерит меню **из локального буфера коуча** (всегда видно ваши последние правки, даже неопубликованные). На устройствах участников этого флага нет — они видят только `data.json.coach_overrides` (опубликованную версию).

## Локальный запуск

PWA требует HTTP — открытие через `file://` сработает (есть `window.fallbackData`), но Service Worker и `manifest` корректно подхватятся только с сервера.

```bash
# Python 3 (есть на любой ОС)
python -m http.server 8000

# Или Node без зависимостей
npx serve .
```

Затем откройте `http://127.0.0.1:8000/` в браузере. На телефоне через DevTools/Lighthouse приложение можно «Установить на главный экран».

## Деплой на GitHub Pages

1. Запушьте проект в репозиторий (например, `main`).
2. В настройках репозитория → **Settings → Pages** выберите ветку `main`, папку `/ (root)`.
3. Через минуту приложение будет доступно по адресу вида `https://<user>.github.io/Diet-journal/`.

Service Worker заработает автоматически, потому что Pages отдаёт всё по HTTPS.

## Как коуч обновляет меню/список

1. Открыть `data.json` (можно прямо на GitHub в вебе → **Edit this file**).
2. Изменить или добавить недели в массиве `weeks` — структура задокументирована ниже.
3. Сохранить (commit). У всех участников меню обновится при следующем открытии: `fetch('data.json?...')` использует cache-bust, а Service Worker заменит кэш через стратегию network-first.

### Формат `data.json`

```json
{
  "marathon": {
    "title": "Марафон −10",
    "total_days": 21,
    "start_date": "2025-06-01"
  },
  "weeks": [
    {
      "week": 1,
      "menu": {
        "Понедельник": {
          "breakfast": "Овсяная каша с ягодами",
          "lunch":     "Куриный суп",
          "dinner":    "Греческий салат"
        }
      },
      "shopping": ["Куриное филе 1 кг", "Овсянка 500 г"]
    }
  ]
}
```

Имена дней недели должны быть строго `Понедельник`, `Вторник`, … `Воскресенье`.

> **`marathon.start_date`** — это дата по умолчанию.
> Каждый участник на своём устройстве получает **личную** дату старта (`marathon_start` в `localStorage`), равную дню, когда он впервые открыл приложение. Поэтому «День 1 из 21» всегда отсчитывается от первого открытия, а не от даты в `data.json`.

### Расширенный формат блюда (раскрывающаяся карточка)

Любое поле `breakfast` / `lunch` / `dinner` можно записать как **строку** (компактный режим) или как **объект** — тогда блюдо станет раскрывающейся карточкой с составом, рецептом и КБЖУ:

```json
"Понедельник": {
  "breakfast": {
    "title": "Овсяная каша с ягодами",
    "ingredients": [
      "Овсяные хлопья — 50 г",
      "Молоко 1,5% — 200 мл",
      "Ягоды — 80 г",
      "Мёд — 1 ч.л."
    ],
    "recipe": "Залить овсянку молоком, варить 5–7 минут. Добавить мёд и ягоды.",
    "kbju": "Б 9 г · Ж 8 г · У 55 г",
    "calories": 340
  },
  "lunch":  "Куриный суп",
  "dinner": "Греческий салат"
}
```

Все поля внутри объекта необязательны:

| Поле | Тип | Что делает |
| --- | --- | --- |
| `title` | string | Заголовок блюда в свёрнутом виде |
| `ingredients` | string[] | Маркированный список «Состав» |
| `recipe` | string | Текст рецепта |
| `kbju` | string | Подпись с КБЖУ, например `"Б 9 г · Ж 8 г · У 55 г"` |
| `calories` | number | Показывается рядом с КБЖУ |

Если в блюде есть хоть что-то кроме `title` — рядом появляется стрелочка-шеврон, и пользователь может тапнуть, чтобы раскрыть подробности.

## Хранение данных участника (localStorage)

| Ключ | Значение |
| --- | --- |
| `user_name` | строка, имя участника |
| `marathon_start` | строка `YYYY-MM-DD`, личная дата старта марафона (день первого открытия приложения) |
| `day_YYYY-MM-DD` | JSON `{ date, weight, steps, waist, hips, chest }` |
| `coach_overrides` | JSON `{ "YYYY-MM-DD": { breakfast?, lunch?, snack?, dinner? } }` — локальный буфер коуча |
| `coach_mode_active` | строка `"1"`, выставляется при первом успешном входе в режим коуча на устройстве |
| `gh_token` | строка, GitHub Personal Access Token (только на устройстве коуча, **не экспортируется**) |
| `gh_last_publish` | строка — время и SHA последней публикации |
| `shopping_extra_week_N` | JSON-массив строк (личные добавления к списку недели N) |
| `shopping_checked_week_N` | JSON-массив **строк** — отмеченные пункты недели N (по тексту, не по индексу) |

Ключи `gh_token`, `coach_mode_active`, `gh_last_publish` исключены из экспорта/импорта JSON-бэкапа — токен и флаг коуча не должны утечь к другим участникам.

Все данные участника никогда не покидают устройство. Бэкап и перенос — через кнопки «Скачать JSON» / «Импортировать JSON» в **Архиве**.

## Регенерация иконок

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\make-icons.ps1
```

Скрипт перерисует `icons/icon-192.png` и `icons/icon-512.png` в фирменных цветах.

## Стек

- Чистые HTML / CSS / JS, без сборки и зависимостей.
- SVG-график и иконки нарисованы вручную (без Chart.js / иконочных шрифтов).
- Цветовая схема: `#1a1a2e` (тёмный) и `#e94d6b` (акцент).
